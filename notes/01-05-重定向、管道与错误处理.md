在Shell环境中，几乎所有命令都遵循「**标准输入**→**标准输出**→**标准错误**」这一模型：

> 链式处理（装饰器模式）
> 
- 使用>、>>、<、<<可以将输入输出重定向到文件或从文件读入；
- 使用管道符号|可以把一条命令的输出实时作为下一条命令的输入；
- 使用2>这样的写法可以将错误信息单独收集或丢弃（/dev/null）

# 输入输出重定向

- 将输出重定向到指定文件
    - > 覆盖原内容
    - >> 追加到文件末尾
- 输入重定向
    - < 将输入重定向为指定文件的内容
    - << 将后续输入行作为命令的参数，指导遇到指定的标记结束，例如
        
        ```bash
        <<'COMMENT'
        输入参数
        COMMENT
        ```
        

例如像

```bash
命令名 [选项] [参数] < 文件名
```

从指定文件中拿到输入

```bash
命令名 [选项] [参数] > 文件名
```

执行命令写入指定文件

# 错误重定向

命令的正常结果会输出到标准输出中，

像 语法错误、执行错误信息会收集到标准错误中

2> 可以指定标准错误的收集文件 

```bash
命令名 >> out.log 2>> err.log
```

正常输出追加到out.log中，错误输出追加到err.log中

# 管道

符号 | 

前面提到了 shell 是支持链式处理的，管道是用于在命令之间交换数据的通信机制，数据流从管道中像经过流水线一样经过命令

常见场景有

分页查看

```bash
ls | more
```

结果筛选

grep

```bash
ps aux | grep nginx
ls -l | grep ".txt"
```
