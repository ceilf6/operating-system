# Linux 第 5 次习题课（TD n°5）

## 用户账户的自动创建与删除脚本

现有一个名为 `listing` 的文件，其中包含需要在 Linux 系统上创建计算机账户的用户姓名列表。

文件格式如下：

```text
# 第一份名单
Dupont Charles
Martin Paul
# 第二份名单
Retraint Florent
...
```

## 题目要求

### 1.
编写一个 shell 脚本，逐行读取 `listing` 文件，并按如下形式回显到屏幕上：

```text
<姓> <名>
```

### 2.
修改脚本，去掉 `listing` 中可能存在的注释行（即：以 `#` 字符开头的行）。

### 3.
在考虑第 1、2 问要求的基础上，将以下信息显示到屏幕上：

```text
<姓> <名> <小写首字母缩写>
```

使用 `tr` 命令将大写字母转换为小写字母。

### 4.
修改脚本，使 `listing` 文件名通过命令行参数传入，并在继续执行前检查该文件是否存在。

提示信息要求如下：

- 如果没有提供参数，显示：`Usage : ...`
- 如果作为参数传入的文件不存在，显示：`Fichier <….> inexistant`

为了避免登录名重复，约定使用以下规则生成用户登录名（login）：

- 取姓氏的前 2 个字符
- 再取名字的前 2 个字符
- 全部转换为小写

并用该 login 替换第 3 问中的 `<INITIALES>`。

### 5.
使用第 4 问中构造出的 login 来创建账户，`useradd` 命令需使用以下信息（执行该命令需要 root 权限）：

- **login**：按第 4 问定义
- **home 目录**：`/home/NOM_xy`，其中 `xy` 为名字前两个字母的小写形式
- **UID**：第一个账户为 `1000`，之后每创建一个账户递增 `1`
- **用户组**：`users`
- **注释（comment）**：`prénom NOM-TP ADMIN LINUX`
- **shell**：`/bin/bash`
- **密码（passwd）**：先创建一个 `guest` 账户（`useradd guest`）；为其设置密码；然后将 `guest` 的加密密码作为 `useradd` 命令中 `-p` 选项的密码值

### 6.
使用 `su` 命令手动测试这些新建账户是否可以登录，并检查用户目录是否已正确创建。

### 7.
修改脚本，支持一个删除选项 `-del`：该选项应使用 `userdel` 命令删除 `listing` 文件中对应用户的账户，并在每次删除时显示如下信息：

```text
Suppression du compte <....> : [OK]
```

如果删除失败，则显示：

```text
[ERROR]
```

并跟随 `userdel` 输出的错误信息。

不要忘记删除你所创建的用户目录。
