# test

test 是 Shell 的测试命令，用来判断条件是否成立

`test -f` 判断是否为普通文件

`test -d` 判断是否为目录文件

可以将if和then放在同一行，用分号分隔：

```bash
if test -f $1 ; then file $1 ; fi
```

## [ ] 写法

注意 [ ] 两边得有空格

```bash
    if [ -r $1 ];then
    # 等价于 
    if test -r $1;then
```

## 文件属性测试

-f 文件存在且为普通文件

-e 文件存在

-r 文件存在且可读

-w 文件存在且可写

## 字符串测试

-z 字符串长度为0 zero

-n 字符串长度不为0

`字符串1 = 字符串2` 两个字符串相等

`字符串1 != 字符串2` 两个字符串不等

> 注意判断相等、不等的 **空格** ，如果没有的话会影响判断

# && 和 ||

运算符 && 和 || **根据前一条命令的返回码决定是否执行后一条命令。**

- 命令1 &&命令2
    - 等价于： `if命令1 ; then命令2 ; fi`
    - 当命令1返回0（成功）时执行命令2。
- 命令1 ||命令2
    - 等价于： `if命令1 ; then ; else命令2 ; fi`
    - 当命令1返回非0（失败）时执行命令2。

```bash
echo "输出成功" && echo "前面成功就执行"
test -f $testFilePath1 || echo "前面失败就执行"
# file1不是普通文件
<< 'COMMENT'
输出成功
前面成功就执行
前面失败就执行
COMMENT
```

```bash
if test -f "$1" && test -r "$2"
# 等价于
if test -f "$1" -a -r "$2"
```

```bash
if test -f "$1" || test -r "$2"
# 等价于
if test -f "$1" -o -r "$2"
```

# case多路分支

每个模式值以)结尾。若多个模式执行相同操作，用|分隔。

每个命令块**以 ;; 结尾**。默认分支使用 *)，且必须放在最后。

```bash
# case分支处理
testFilePath4=$(pwd)/test-files/script2.sh
touch "$testFilePath4"
echo "是否要删除文件 $testFilePath4 ?(y删除 n不删)"
read input
case $input in
    y|Y)
        echo "删除文件 $testFilePath4"
        rm "$testFilePath4"
        ;;
    n|N)
        echo "不删除文件 $testFilePath4"
        ;;
    *)
        echo "输入错误，请输入 y/Y 或 n/N"
        ;;
esac
```

> 每个区块的结束标识都是 **开始标识的翻转**
>

## until

```bash
input=""
until [ ! "$input" != "exit" ]
# 注意空格
do
    echo "$input" >> "$outputFilePath"
    echo "请输入内容, 输入exit退出"
    read input
done
```
