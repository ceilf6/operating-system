文件管理、权限管理、挂载方式、备份

文件系统是存储介质（硬盘、软盘、光盘等）上组织和存储目录树结构的一种方式

每个专有操作系统都有自己的文件组织方式

可以在同一块硬盘的不同分区中并存多个文件系统

Linux有自己的文件系统，如 ext2, ext3, ext4 ，同时也能管理其他文件系统，支持的列表可以在

```tsx
/proc/filesystem
```

中查看

在 Unix 体系中，对用户而言一切皆文件，无论物理位置如何，所有文件系统都必须整合进 **Linux 系统唯一的逻辑目录树中**

# 文件类型

## 普通文件

内容保存在磁盘中

- 文本文件：程序源码、脚本、配置文件
- 可执行文件：二进制代码程序

## 目录文件

在 NodeJS 中学过的：目录本质就是文件

作为容器文件，包含对其他文件的引用，是目录树的骨架，用于对文件进行分类管理

## 设备文件

位于 /dev 目录下面，是系统为设备提供的访问入口。挂载操作会将这些特殊文件与它们的挂载点目录建立对应关系。例如：

- /dev/sda：访问和加载第一块IDE硬盘
- /dev/tty：终端设备
- /dev/null：一个“黑洞”设备，写进去的数据会被丢弃
- /dev/random：随机数设备

本质是内核给设备提供的**访问入口**，就像“函数式编程”的思想

程序对设备文件的读写操作本质就是在和硬件或者内核驱动打交道

### 类型

1. 字符设备文件，以流式读取
    - 终端
    - 串口
    - /dev/null
2. 块设备文件，块式读取
    - 硬盘
    - U盘
    - 分区

```tsx
ls -l /dev
```

拿到的一系列文件，看最左边是 c 的就是字符(chr)
是 b 的就是块设备(block)

```tsx
crw-rw-rw- 1 root root 1, 3 ... /dev/null
brw-rw---- 1 root disk 8, 0 ... /dev/sda
```

同一个命令如 cat 对 dev 中的文件实际操作都是不一样的

# 文件操作

```bash
# NodeJS/sandboxs/fs 对照 NodeJS 中的文件操作

# 沙盒环境
cd ./test-files

# 创建文件
touch file1
echo 1 >> file1

# 复制文件
cp ./file1 ./file1-copy

# 删除文件
rm ./file1

# 修改文件名
mv ./file1-copy ./重命名后文件名
mv ./重命名后文件名 ./esN

# 查看文件内容
cat ./esN
more ./esN # more 支持分页查看: 按回车翻行、按空格翻页

# 查看文件类型
file ./esN
# ./esN: ASCII text

cp ./esN ./esN2
echo 2 >> ./esN2

# 比较文件差异
## cmp 二进制逐字节比较，如果完全一致就什么都不输出
cmp ./esN ./esN2 
# cmp: EOF on ./esN

## diff 逐行比较，适合比较文本内容如代码的差异
diff ./esN ./esN2
# 1a2
# > 2

## comm 查看两个文件哪些行是独有的、哪些行是共有的
comm ./esN ./esN2
#                 1
#         2

# 退出沙箱
cd ../
```

# 权限

上面 ls -l 查看了文件的详细信息后，可以看到权限相关的信息

第一个字段中除了第一列代表文件类型外的剩余9个字母

### 对象

其中三个为一段区分为三段从左到右分别表示
u(user)所有者权限 g(group)所属组权限 o(other)其他用户权限
如第一个就是
rw- r-- r--

### 权限类型

e excute 可执行
w write 可写
r read 可读

### 权限表示

和 React 中的权限设置一样，都是用的位运算
0 无任何权限
1 2**0 执行权限
2 2**1 写入权限
4 2**2 读取权限

```bash
chmod 520 esN
# 5=4+1读+可执行 r-x 
# 2写 -w- 
# 0无权限 ---
# -r-x-w----

chmod o+w esN
# -r-x-w--w-
# u(user)所有者权限 
# g(group)所属组权限 
# o(other)其他用户权限

# + 添加
# - 移除
# = 覆盖

# - 无权限
# x 可执行
# w 可写
# r 可读取

chmod g=rx esN
# -r-xr-x-w-

ls -l ./
```

```bash
# 将用户主目录设置为私有目录（仅自己可访问）
chmod 700 -
```

## 硬链接

```tsx
ln 源文件 链接文件
```

指向文件的**索引**，而不是路径

删除原来的文件名不影响硬链接访问数据

不能跨文件系统

不能对目录创建硬链接

## 软链接

```tsx
ls -s 源文件 链接文件
```

得用 -s 参数声明是 soft 软链接

> 也称为符号链接
> 

是一个特殊**文件**，存储另一个文件的**路径**，访问软链接时系统会自动解析到目标路径

**避免了文件在磁盘上的重复拷贝**

类似于快捷方式，指向文件路径

如果目标文件删除，软链接就会失效（悬挂链接）

可以对目录创建软链接

ls -l 可以看到链接文件信息

## 包管理器npm yarn中的link就是软链接

首先在目标包目录中用 npm link 创建软链接

然后在目标使用目录中用 npm link 目标包目录名

接着在目标使用目录中直接 require 目标包目录名即可

## pnpm中用硬链接

像 npm / yarn 在 install 之后会将目标包目录中所有文件都复制一份到自己的 node_modules ，不同项目的 node_modules 是独立的副本，会导致有重复的内容

但是 pnpm 是在有一个 全局的store ，在安装依赖时依赖的内容只在store存储一次，在各个node_modules下面通过 硬链接 指向store

让 **硬链接** 负责 **全局store** 实现**存储**层面共享内容

**软链接** 负责 项目目录的依赖路径解析，让 Node 可以**找**到包
