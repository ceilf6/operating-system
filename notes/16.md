## 自动识别系统中开机自启动的服务

要识别你的 Unix 系统中会自动启动的服务：

```bash
systemctl list-unit-files --type=service | grep enabled
```

或者：

```bash
chkconfig --list
```

禁用某个服务，使其不再自动启动：

```bash
systemctl disable apache2
```

或者：

```bash
update-rc.d apache2 disable
```

确定系统当前的运行级别：

```bash
runlevel
```

将运行级别切换到最小使用模式（单用户模式）：

```bash
init 1
```

掌握关闭和重启 Unix 系统的命令：

```bash
reboot
shutdown -h now
```

计划在 15 分钟后关闭系统：

```bash
shutdown -h +15
```

取消已计划的关机：

```bash
shutdown -c
```

停止和启动服务。

## 练习 2 —— 使用脚本实现自动化

创建一个脚本，在系统启动后检查所有关键服务的状态。

1. 编写一个 bash 脚本，用于：
    - 检查 nginx、mysql 和 ssh 服务的状态。
    - 将结果记录到一个日志文件中。
2. 配置该脚本在系统启动时执行。

### 参考答案

编写脚本 `check_services.sh`：

```bash
#!/bin/bash
services=("nginx" "mysql" "ssh")
log_file="/var/log/services_status.log"
echo "Service Status Report - $(date)" > $log_file
for service in "${services[@]}"; do
 # 对于 SysVinit
 status=$(service $service status 2>/dev/null | grep -q "running" && echo "active" || echo "inactive")
 # 或者
 # status=$(initctl status $service 2>/dev/null | grep -q "running" && echo "active" || echo "inactive")
 echo "$service: $status" >> $log_file
done
```

让脚本可执行：

```bash
chmod +x /usr/local/bin/check_services.sh
```

添加如下行，以便在系统启动时执行该脚本：

```bash
@reboot /usr/local/bin/check_services.sh
```

## 练习 4 —— 归档并压缩文件

学习使用 `tar` 和 `gzip` 对文件进行归档与压缩。

1. 创建一个名为 `projets.tar` 的归档文件，包含 `scripts` 和 `docs` 两个目录。
2. 将该归档压缩为 `projets.tar.gz`。
3. 将压缩归档解压到名为 `backup` 的目录中。

### 参考答案

```bash
tar -cf projets.tar scripts docs      # 创建归档文件 "projets.tar"
gzip projets.tar                      # 压缩为 "projets.tar.gz"
mkdir backup                          # 创建目录 "backup"
tar -xzf projets.tar.gz -C backup     # 解压到 "backup"
```

## 练习 7 —— 在 Linux 下创建和管理分区

在磁盘上创建、格式化、挂载一个新分区，并配置系统开机时自动挂载。

### 参考答案

### 创建新分区

使用以下命令列出磁盘及其分区：

```bash
lsblk
```

创建一个新分区：

```bash
fdisk /dev/sdb1
```

进入 `fdisk` 后：

- 输入 `n` 创建新分区。
- 选择分区类型（主分区或扩展分区）。
- 指定分区大小（例如 `+10G` 表示创建一个 10 GB 的分区）。
- 输入 `w` 保存修改并退出 `fdisk`。

检查新分区是否已创建：

```bash
lsblk
```

### 格式化分区

将新分区格式化为 `ext4` 文件系统：

```bash
mkfs ext4 /dev/sdb1
```

检查文件系统：

```bash
blkid /dev/sdb1
```

### 挂载分区

创建挂载目录（例如 `/mnt/data`）：

```bash
mkdir -p /mnt/data
```

将分区挂载到该目录：

```bash
mount /dev/sdb1 /mnt/data
```

检查分区是否已挂载：

```bash
df -h
```

### 配置自动挂载

编辑 `/etc/fstab` 文件：

```bash
nano /etc/fstab
```

在文件末尾添加如下行：

```bash
/dev/sdb1 /mnt/data ext4 defaults 0 2
```

无需重启即可测试配置：

```bash
mount -a
```

## 练习 8 —— 启用和停用 swap 文件

swap 不仅可以通过 swap 分区来管理，也可以通过 swap 文件来管理。说明如何创建、启用和停用 swap 文件。

### 参考答案

创建一个 swap 文件（例如 1 GB）：

```bash
dd if=/dev/zero of=/swapfile bs=1M count=1024
```

说明：

- `if` = Input file（输入文件）
- `of` = Output file（输出文件）
- `bs` = Block size（块大小）
- `count` = 块数量

设置正确权限：

```bash
chmod 600 /swapfile
```

将该文件配置为 swap 空间：

```bash
mkswap /swapfile
```

启用 swap：

```bash
swapon /swapfile
```

检查 swap 是否已启用：

```bash
swapon -s
```

当不再需要时停用 swap：

```bash
swapoff /swapfile
```

删除 swap 文件：

```bash
rm /swapfile
```

## 练习 9 —— 启用和停用 swap 分区

swap 可以通过 swap 文件或 swap 分区来管理。说明如何创建、启用和停用 swap 分区。

### 参考答案

使用 `fdisk` 在磁盘上创建一个分区：

```bash
fdisk /dev/sdb
```

然后：

- 输入 `n` 创建新分区。
- 输入 `t` 并选择类型 `82`（Linux swap），将其设置为 Linux swap 分区。
- 输入 `w` 保存。

将该分区格式化为 swap：

```bash
mkswap /dev/sdb1
```

启用 swap 分区：

```bash
swapon /dev/sdb1
```

检查 swap 分区是否已启用：

```bash
swapon --show
```

将该 swap 分区添加到 `/etc/fstab`，使其在系统启动时自动启用：

```bash
/dev/sdb1 none swap sw 0 0
```

## 练习 10 —— Unix/Linux 下的网络配置

学习如何在 Unix/Linux 下配置和诊断网络参数，使用网络管理工具和命令。给出用于检查网络配置、配置静态 IP 地址以及配置 DNS 服务器的命令。

### 参考答案

### 检查当前网络配置

显示网络接口及其 IP 地址：

```bash
ip a
```

或者：

```bash
ifconfig
```

显示网络路由和默认网关：

```bash
ip route show
```

### 配置静态 IP 地址

打开 `/etc/network/interfaces` 文件进行编辑：

```bash
nano /etc/network/interfaces
```

静态 IP 配置示例：

```bash
auto eth0
iface eth0 inet static
address 192.168.1.100
netmask 255.255.255.0
gateway 192.168.1.1
```

保存文件后，重启网络接口：

```bash
systemctl restart networking
```

### 配置 DNS 服务器

修改 `/etc/resolv.conf` 文件，添加 DNS 服务器：

```bash
nano /etc/resolv.conf
```

例如加入以下内容：

```bash
nameserver 8.8.8.8
nameserver 8.8.4.4
```

## 练习 13 —— 配置静态路由

1. 添加一条静态路由，使所有发往网络 `192.168.2.0/24` 的流量都通过网关 `192.168.1.1`。
2. 检查该路由是否已添加。

### 参考答案

```bash
ip route add 192.168.2.0/24 via 192.168.1.1
```

或者：

```bash
route add -net 192.168.2.0 netmask 255.255.255.0 gw 192.168.1.1
```

检查路由：

```bash
ip route
```

或者：

```bash
route -n
```
